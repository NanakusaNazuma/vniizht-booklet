# Буклет ВНИИЖТ — QR → листаемый буклет на телефоне

## Короткий ответ

**Да.** После публикации на Vercel (или другом хостинге) **любой человек** с телефоном
и интернетом сможет:

1. Навести камеру на QR-код  
2. Открыть буклет в браузере  
3. Листать страницы свайпом  

Wi‑Fi вашего компьютера **не нужен** — нужен только интернет на телефоне.

---

## Как это работает

```
QR на бумаге  →  https://ваш-сайт.vercel.app/  →  буклет на весь экран
```

- `/` — буклет (ссылка в QR)
- `/booklet` — старые QR; автоматически перенаправляет на `/`
- `/qr` — страница, где скачать QR-картинку для печати

---

## Публикация через GitHub + Vercel (бесплатно)

### Шаг 1. Загрузить проект на GitHub

В терминале VS Code:

```bash
cd D:\WebLogisticVNIIZHT
git init
git add .
git commit -m "Буклет ВНИИЖТ с QR"
```

На [github.com](https://github.com) создайте **новый репозиторий** (без README).
Затем:

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/vniizht-booklet.git
git branch -M main
git push -u origin main
```

### Шаг 2. Подключить Vercel

1. Зайдите на [vercel.com](https://vercel.com) → войти через **GitHub**
2. **Add New… → Project**
3. Выберите репозиторий `vniizht-booklet`
4. Vercel сам определит **Vite** — ничего менять не нужно:
   - Build: `npm run build`
   - Output: `dist`
5. Нажмите **Deploy**

Через 1–2 минуты получите адрес вида:

`https://vniizht-booklet.vercel.app`

### Шаг 3. Скачать QR для печати

1. Откройте в браузере: `https://vniizht-booklet.vercel.app/qr`
2. Адрес подставится автоматически
3. Нажмите **«Скачать QR для печати»**
4. Вставьте картинку на бумажный буклет

### Шаг 4. Проверить с телефона

Отсканируйте QR камерой — должен открыться буклет и листаться свайпом.

## Обновление буклета

Контент страниц хранится в **`public/booklet-slides.pdf`** (векторный экспорт из PowerPoint).
Редактировать удобнее **исходник** `public/booklet.pptx`, а не PDF напрямую.

### Как править текст и вёрстку (страницы 4, 7 и др.)

1. Откройте **`public/booklet.pptx`** в **Microsoft PowerPoint** (или LibreOffice Impress).
2. Перейдите на нужный слайд (слайд 4 = страница 4 в буклете).
3. Исправьте текст:
   - **Стр. 4:** «АО» и ««ВНИИЖТ»» — выделите оба фрагмента и сделайте **одним текстовым блоком** на одной строке (или уменьшите шрифт / расширьте рамку, чтобы не переносилось).
   - **Стр. 7:** «Задачи проекта» — увеличьте ширину текстового поля или уменьшите кегль, чтобы слово **«проекта»** не разрывалось (буква «а» на отдельной строке = слишком узкое поле).
4. **Файл → Сохранить как → PDF** (или «Экспорт → Создать PDF»).
5. Сохраните PDF как **`public/booklet-slides.pdf`** (замените старый файл).
6. Если меняли 2-й слайд (письмо с фигурой) — обновите **`public/person.png`** (экспорт фигуры из PPTX отдельно, как сейчас).

> PDF как «рисунок» править сложно. PowerPoint — единственный нормальный способ менять текст и переносы строк.

### Залить изменения на GitHub и Vercel

В терминале в папке проекта:

```bash
cd D:\WebLogisticVNIIZHT
git add public/booklet-slides.pdf public/booklet.pptx
git add public/person.png   # только если меняли 2-й слайд
git commit -m "Обновление текста буклета"
git push
```

Vercel, привязанный к репозиторию, **сам пересоберёт сайт** за 1–2 минуты.
QR-код менять **не нужно**, если адрес сайта тот же (`vniizht-booklet-ntfr.vercel.app`).

Проверка: откройте сайт на телефоне в режиме инкогнито или обновите страницу с очисткой кэша.

## Мобильный просмотр и увеличение

На телефоне:

- **Свайп** — листать страницы
- **Щипок двумя пальцами** или кнопки **+/−** в углу страницы — увеличить (до 300%)
- **Двойное нажатие** — быстро 100% ↔ 200%

Удобно для графиков на 5-й странице и мелкого текста.
